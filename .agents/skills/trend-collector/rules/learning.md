# 興味学習（Interest Learning）ルール

ユーザーが「この記事面白い」「この記事興味ある」等のメッセージとともに記事URLを渡した場合の処理手順です。

## 学習手順
1. 渡された記事URLを `scripts/fetch_article.sh` で取得し本文を読みます。
   `bash scripts/fetch_article.sh "<記事URL>" 8000`
2. 記事の主題を分析し、`config.jsonc` の `interest_profile.rating_criteria` のどの興味軸に該当するか、または新しい興味領域かを判断します。
3. `config.jsonc` の `learned_signals` 配列に以下の形式で追記します：
   `"YYYY-MM-DD | トピック名 — 注目した理由の簡潔な説明"`
4. 追記後「✅ 興味シグナルを学習しました：`トピック名`」とユーザーに通知します。

## 淘汰ルール
- `learned_signals` が **20件を超えた場合**、配列の先頭（最も古い）から削除して20件以内に維持します。
- 既存のシグナルと**重複するトピック**の場合は、日付を更新して既存エントリを置き換えます（同じシグナルが2つにならないようにします）。
