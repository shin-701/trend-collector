# エラーレポートフォーマット

エラー発生時に `.agents/skills/trend-collector/logs/YYYY-MM-DD_error.md` へ出力するテンプレート。

> 💡 `logs/` ディレクトリが存在しない場合は自動で作成する。

## テンプレート

```markdown
---
created: YYYY-MM-DD
type: error-report
skill: trend-collector
---

# Trend Collector エラーレポート: YYYY-MM-DD

## エラー概要

| 項目 | 内容 |
|------|------|
| 発生日時 | YYYY-MM-DD HH:mm:ss |
| 発生ステップ | ステップ X. （ステップ名） |
| エラー種別 | （ネットワーク / パース / 設定 / その他） |

## エラー詳細

### エラー 1: （エラーの簡潔な説明）

- **発生箇所**: （スクリプト名や処理の場所）
- **エラーメッセージ**: （実際のエラー出力）
- **原因の推定**: （考えられる原因を1〜2文で）
- **対応策**:
  1. （具体的な対応手順）
  2. （代替手段があれば）

## 実行結果サマリ

| ソース | 状態 | 取得件数 |
|--------|------|----------|
| はてなブックマーク | ✅ 成功 / ❌ 失敗 | XX件 |
| Hacker News | ✅ 成功 / ❌ 失敗 | XX件 |
| Reddit | ✅ 成功 / ❌ 失敗 | XX件 |

## 備考

（部分的に成功した場合の補足情報、次回実行時の注意点など）
```

## エラー発生時の動作ルール

- **部分的な成功**: 一部のソースだけ失敗した場合は、成功したソースの結果で `neta-trend.md` を生成し、エラーレポートも別途出力する
- **全面的な失敗**: すべてのソースが失敗した場合は `neta-trend.md` を生成せず、エラーレポートのみ出力する
- **同日の既存レポート**: 同日にエラーレポートが既に存在する場合は、内容を追記（上書きしない）
- **チャットへの通知**: エラー発生時は「⚠️ 一部エラーが発生しました。詳細: `.agents/skills/trend-collector/logs/YYYY-MM-DD_error.md`」とユーザーに通知する
