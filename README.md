# Trend Collector

個人に最適化された技術トレンド収集・分析・レポーティングシステムです。
はてなブックマーク、Hacker News、Reddit から情報を収集し、LLM（Antigravity / Claude）を用いて個人の興味関心に基づいたフィルタリングと要約を行います。

## 特徴

- **マルチソース収集**: 
  - **はてなブックマーク**: 国内の IT・テックトレンド
  - **Hacker News**: 海外の深掘り技術・スタートアップトレンド
  - **Reddit**: 特定コミュニティ（AI, PKM, キャリア等）の議論
- **インテリジェント・フィルタリング**:
  - 単なるキーワードマッチではなく、`config.jsonc` に定義された「興味プロファイル」に基づき、LLM が文脈を理解して 3 段階（★★★, ★★, ★）で評価。
- **自動レポーティング**:
  - 主要なトレンド記事（5〜7 件）を日本語 5 行要約。
  - 全収集記事（90 件以上）を自動的に日本語翻訳し、メタデータ付きのテーブル形式で出力。
- **興味学習機能**:
  - 「この記事面白い」というフィードバックを LLM に与えることで、将来の評価基準を自動的にアップデート。

## アーキテクチャ

このプロジェクトは、高速なデータ収集を行う **Shell/Python スクリプト** と、高度な判断・要約を行う **LLM (Antigravity Skill)** のハイブリッド構成になっています。

- **データ収集**: `curl` や `grep` を用いた並列スクレイピング（`scripts/*.sh`）
- **データ加工**: Python によるパース、翻訳、テーブル整形（`scripts/*.py`）
- **意思決定**: Antigravity スキルによる評価・要約・レポーティング

## 使い方

### Antigravity スキルとして実行（推奨）

Antigravity（または対応する AI エージェント）に対し、以下のように話しかけるだけで全工程が自動実行されます。

- 「今日のトレンドを教えて」
- 「ネタ収集して」
- 「最近の AI 関連の話題は？」

### 自分のローカルのAIエージェントにスキルを設定する場合

claudeCodeでスキルを設定する場合は、ディレクトリ構成を変更する必要があります。
`.agents/skills/trend-collector`ディレクトリを`.claude/skills/trend-collector`に移動してください。

## 設定

`.agents/skills/trend-collector/config.jsonc` で動作をカスタマイズできます。

- `interest_profile`: 興味度の判定基準（★★★ 〜 ★）を定義します。
- `thresholds`: 各ソースでの収集対象とする最低ライン（はてブ数、HN ポイント数、Reddit Ups 数）を設定します。
- `sources`: 収集対象の URL やサブレディットを管理します。

## ディレクトリ構造

```text
.
├── .agents/
│   └── skills/
│       └── trend-collector/
│           ├── SKILL.md          # スキルの定義・実行手順
│           ├── config.jsonc      # メイン設定ファイル
│           ├── rules/            # LLM 用の動作ルール
│           ├── scripts/          # 収集・加工用スクリプト群
│           └── templates/        # 出力フォーマット定義
└── (output)/
    └── 11_ideas/logs/            # 生成されたログやレポートの出力先
```

## 依存関係
